## 설치

Biome을 다운로드하는 가장 빠른 방법은 `npm`과 같은 패키지 관리자를 사용하는 것입니다.
이를 위해서는 Node.js v14.18 이상이 필요합니다.
Node.js를 설치하지 않고 Biome을 사용하려는 경우 [단독 실행 파일](/ko/guides/manual-installation)로 CLI를 사용할 수 있습니다.

Biome을 설치하려면 `package.json` 파일이 포함된 디렉터리에서 다음 명령을 실행하세요.

<PackageManagerCommand
  npm="install --save-dev --save-exact @biomejs/biome"
  pnpm="add --save-dev --save-exact @biomejs/biome"
  yarn="add --dev --exact @biomejs/biome"
  bun="add --dev --exact @biomejs/biome"
  deno="add --dev npm:@biomejs/biome"
/>

:::note
패키지 관리자에게 정확한 버전의 Biome을 고정하도록 지시합니다.
이렇게 하면 프로젝트 내의 모든 사람이 정확히 동일한 버전의 Biome을 사용할 수 있습니다.
패치 릴리스조차도 약간 다른 동작을 초래할 수 있기 때문입니다.
자세한 내용은 [버전 관리](/ko/internals/versioning/) 페이지를 참조하세요.
:::


## 구성 설정

각 프로젝트에 대해 `biome.json` 또는 `biome.jsonc` 구성 파일을 만드는 것이 좋습니다.
이렇게 하면 명령을 실행할 때마다 CLI 옵션을 반복할 필요가 없으며,
Biome이 편집기에서 동일한 구성을 사용하도록 보장할 수 있습니다.
일부 옵션은 구성 파일에서만 사용할 수 있습니다.
Biome의 기본값에 만족하는 경우 구성 파일을 만들 필요가 없습니다.
`biome.json` 파일을 만들려면 프로젝트의 루트 폴더에서 `init` 명령을 실행하세요.

<PackageManagerBiomeCommand command="init"/>

대신 `biome.jsonc` 파일을 만들려면 `--jsonc` 옵션을 사용하세요.

`init` 명령을 실행하면 디렉터리에 새 `biome.json` 파일이 생깁니다.

<DefaultConfiguration/>

`linter.enabled: true`는 린터를 활성화하고 `rules.recommended: true`는 [권장 규칙](/ko/linter/rules/)을 활성화합니다. 이는 기본 설정에 해당합니다.

포매팅은 **기본적으로** 활성화되어 있지만 `formatter.enabled: false`를 사용하여 명시적으로 [비활성화](/ko/reference/configuration/#formatterenabled)할 수 있습니다.


## 사용

Biome CLI에는 다양한 명령과 옵션이 제공되므로 필요한 대로 사용할 수 있습니다.

`--write` 옵션과 함께 [`format`](/ko/reference/cli#biome-format) 명령을 사용하여 파일과 디렉터리를 포매팅할 수 있습니다.

<PackageManagerBiomeCommand command="format --write <파일>"/>

`--write` 옵션과 함께 [`lint`](/ko/reference/cli#biome-lint) 명령을 사용하여 파일과 디렉터리를 린팅하고 안전한 수정 사항을 적용할 수 있습니다.

<PackageManagerBiomeCommand command="lint --write <파일>"/>

[`check`](/ko/reference/cli#biome-check) 명령을 활용하여 두 가지를 **모두** 실행할 수 있습니다.

<PackageManagerBiomeCommand command="check --write <파일>"/>

`check` 명령은 여러 도구를 한꺼번에 실행합니다.
포매팅하고, 린팅하고, import 서식을 지정합니다.


## 에디터 플러그인 설치

Biome을 최대한 활용하려면 에디터 플러그인을 설치하는 것이 좋습니다.
Biome을 지원하는 에디터를 확인하려면 [에디터 페이지](/ko/guides/integrate-in-editor)를 확인하세요.


## CI 설정

Node.js를 사용하는 경우 CI에서 Biome을 실행하는 권장 방법은 [선호하는 패키지 관리자](/ko/guides/getting-started#installation)를 사용하는 것입니다.
이렇게 하면 CI 파이프라인에서 에디터 내부 또는 로컬 CLI 명령을 실행할 때와 동일한 버전의 Biome을 사용할 수 있습니다.
아니면 전용 [CI 동작](/ko/recipes/continuous-integration)을 사용할 수도 있습니다.
