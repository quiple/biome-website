Biome은 모노레포나 여러 프로젝트가 포함된 작업 공간과 같은 대규모 프로젝트에서 적절하게 사용할 수 있는 몇 가지 도구를 제공합니다.


## 여러 개의 구성 파일 사용

CLI 또는 LSP를 통해 Biome의 기능을 사용할 때 Biome은 현재 작업 디렉터리를 사용하는 가장 가까운 구성 파일을 찾습니다.

Biome이 구성 파일을 찾지 못하면 찾을 때까지 파일 시스템의 디렉터리를 **거슬러 올라갑니다**.

이 기능을 활용하여 프로젝트/폴더에 따라 다른 설정을 적용할 수 있습니다.

백엔드 앱과 새로운 프론트엔드 앱이 포함된 프로젝트가 있다고 가정해 보겠습니다.

<FileTree>
  - app
    - backend
      - biome.json
      - package.json
    - frontend
      - biome.json
      - legacy-app
        - package.json
      - new-app
        - package.json
</FileTree>

즉, `app/backend/package.json` 파일에서 스크립트를 실행하면 Biome은 구성 파일 `app/backend/biome.json`을 사용합니다.

`app/frontend/legacy-app/package.json` 또는 `app/frontend/new-app/package.json`에서 스크립트를 실행하면 Biome은 구성 파일 `app/frontend/biome.json`을 사용합니다.


## 구성 설정 공유

[`extends`](/ko/guides/configure-biome/#구성-파일-공유) 구성 옵션을 사용하여 파일 전체에 걸쳐 옵션을 세분화할 수 있습니다.

다음과 같은 요구 사항이 있다고 가정해 보겠습니다.
- `legacy-app`은 공백을 사용하여 포매팅해야 합니다.
- `backend`와 `new-app`은 탭을 사용하여 포매팅해야 합니다.
- 모든 앱은 줄 너비 120으로 포매팅해야 합니다.
- `backend` 앱에 추가 린팅이 필요합니다.

먼저 `app/biome.json`에 새 구성 파일을 만들고 여기에 공유 옵션을 넣습니다.

```json title="app/biome.json"
{
  "formatter": {
    "enabled": true,
    "lineWidth": 120
  }
}
```

이제 다른 포매팅을 사용해야 하므로 `app/frontend/biome.json`을 `app/frontend/legacy-app/`로 **이동**합니다.

```json title="app/frontend/legacy-app/biome.json"
{
  "formatter": {
    "indentStyle": "space"
  }
}
```

그런 다음 `extends` 속성을 사용하여 기본 `app/biome.json` 파일에서 모든 옵션을 상속하도록 Biome에 지시합니다.

```json title="app/frontend/legacy-app/biome.json" ins={2}
{
   "extends": ["../../biome.json"],
   "formatter": {
     "indentStyle": "space"
   }
}
```

린팅을 활성화해야 하는 `app/backend/biome.json`으로 이동합니다.

```json title="app/backend/biome.json"
{
  "extends": ["../biome.json"],
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true
    }
  }
}
```


## 모노레포

모노레포는 여러 개의 라이브러리가 하나의 큰 저장소에 저장되고 유지 관리되는 특정 저장소를 뜻합니다. 각 라이브러리는 서로 다른 구성을 포함할 수 있는 독립된 프로젝트를 나타냅니다.

중첩된 구성 파일을 처리하는 데 몇 가지 제한 사항으로 인해 Biome은 모노레포를 잘 지원하지 않으므로 [관련 이슈를 팔로우하고 도와주세요](https://github.com/biomejs/biome/issues/2228).

현재의 제한에도 불구하고 최상의 개발자 경험을 위해서는 모노레포의 루트에 `biome.json`을 두고 [`overrides`](/ko/reference/configuration/#overrides) 구성을 사용하여 특정 패키지에서 Biome의 동작을 변경하는 것이 좋습니다.

다음 예제에서는 패키지 `packages/logger` 내에서 `suspicious/noConsoleLog` 규칙을 비활성화합니다.

```json title="biome.jsonc"

{
  "linter": {
    "enabled": true,
    "rules": {
      "recommended": true
    }
  },
  "overrides": [{
      "include": ["packages/logger/**"],
      "linter": {
        "rules": {
          "suspicious": {
            "noConsoleLog": "off"
          }
        }
      }
  }]
}
```
